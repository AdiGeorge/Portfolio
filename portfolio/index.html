About

GoSee is a start-up company based in Aarhus which is focused on developing a travel application for mobile phones. However, there is more to the concept than just an ordinary booking app. GoSee is run by a core team of 3 top-up international students - Tomáš Sitárik, Adam Rehák, and Cristian Leahu. Their goal is to create and launch an app that facilitates travel planning by matching the itinerary with the best options for transport and accommodation. Moreover, the element of surprise and innovation involves gamifying the whole trip, thus putting an element of digital play- fulness and fun into real-life traveling. Their idea was born after the founders experienced a series of frustrations caused by the current digital alternatives for travel planning.

We developed a promotional website for the company, showcasing the app's features and concept. Due to a good collaboration and work-flow they decided to use our websolution for promoting the app.

Functionalities

Chest opening

One of the challenges we encountered was making this animation of a chest opening and displaying a modal sign-up form. We achieved this animation by using keyframes and css animations for opening and closing the lid of the chest and jQuey for displaying the modal by adding classes of "opened" and removing them when it's clicked again. We used keyframes and css animations for sliding-up the lid of the chest. Moreover, the chest animation of heartbeat is made with keyframes in order to make it intuitive for clicking.

Animations

In order to make the web solution more attractive and interactive we used a library called Animate on Scroll, which makes the elements sliding in and out when the user scroll the page, and jQuery animations for some icons, making them bigger when hover, clicking on the questions and displaying the answers with a sliding down animation. We made an easy slidieshow for the mobile mock-ups on the front page in order to showcase better the functionalites of the actual app.

Responsiveness

A modern design and a modern website requires responsiveness for all types of devices. Because our clients will use our website for promoting the app, we had to make it responsive. In order to achieve this, we used CSS Grid and Flexbox. We had quite a complicated layout so it was a bit challenging for us (because of the line in the middle, different words on the background, the video and other design elements on top of it).

About

The Web App project was mainly focused on javaScript and fetching the data from a Headless CMS(Flamelink Firebase). We made a movie app where the users can randomly choose a movie if they don't know what to watch. All the data (movie title, description, pictures, reviews) was included in Firebase and then displaying it in the DOM. This project was the most interesting project because we worked a lot with javaScript, we made a lot of functionalities and we developed it as a Single Page Application.I have improved my Frontend Developer skills by diving into some JavaScript concepts in the perspective of multimedia design. We worked with data structures and GitHub.

Functionalities

Fetch

In order to make the web app dynamic we created a database structure with firebase. Here we made a collection named movies where we added movies’ description, images, ratings, basically the whole content and another collection named users where the information the user used for logging in is going to be stored. This way, we can call the content from database using JavaScript and appending it to the DOM. For example, for a specific movie we created a document in firebase and then in that document we created some fields: the answers for the spoilers section, the description of the movie, added the link of the image, rating and the title. In order to run the firebase in our code we had to include some APIs, the links(in the HTML), some variables the configuration for logging in and the function for initializing the script.

Functionalities

Search functionality

We created a search functionality where the user can search his desired movie. We created a search function with “value” as a parameter in order to take the input that the user in typing, even if he writes it in lower case. We made a loop that’s going to loop through the movies collection from database, a variable called title so the user can type the title of the movie (the title is taken from database). The if statement checks if the title includes the search query (the user’s input in lower case). If it’s included, then the searched movie is going to be pushed in the filtered movies array and then it’s going to be appended to appendMovies function.

Randomizer

Our main feature of the app is the randomizer. In order to achieve this, we made a function where we take the movies from the database, count them and then created a function using Math.random() that chooses a random movie based on its id. After selecting the random number, we’ll send the data from firebase to singleMovie(). The singleMovie() function is appending the data based on the id and movie in the DOM. (movie title, picture, description). This is the description page. So if the user clicks on the randomizer it’s going to redirect him to the description page of the movie he found.

Randomizer

In order for the user to log in with email or google we added this constant with the sign in options. We created a constant called ui that is initializing authentication firebase system. The function shows the user when he successfully signs in. For logging out firebase has a built-in method called signOut() and then we are showing the home page.

Single Page Application

We used Single Page Application concept, which is the way you make more pages having only one HTML file. In order to make the transition between pages we had to hide all the pages and choose which is going to be the default page. Then, by selecting the id of the page that we wanted to show up (the id of the div from HTML) we were displaying it block and adding the class “active”. This is really helpful in improving the loading time because it makes only one HTTP request and making the transition between pages smooth.
